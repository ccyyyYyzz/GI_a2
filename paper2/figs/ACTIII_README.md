sentences, no badge boxes/halos/filled component boxes (hairline outlines

names, genuinely needed series legends, in-image PSNR values in panel (d)
(data, Chen-style), guard/threshold lines with terse labels, and the
ruling-mandated development tags on (a)/(b) as small corner tags.

## Data provenance and selection rules

### (a) `actiii_a.pdf` — guard-collapse DEV plot
- **Machinery.** `oed_design_v5.design_v5` → `mixture_search` /
  `path_feasible_alpha` (the frozen PATH_FEASIBLE_ALPHA diagnostic). The
  OED-DT arm that drives this path was retired at R17, so no current campaign
  entry point calls it; the driver was reconstructed from the archived R17
  refreeze selftest (commit `7ba4183`, `m1_freeze_selftest.py`) and run on the
  **development glyph** scene only (`m1_dev_glyph`, seed 632900 class).
- **Cell.** Fast anchor `ν = 2000, b = 0.60`. Inputs: `xhat` =
  `m1_runner.prescan_estimate` of the DEV glyph; load-matched reference =
  deployed SCAT32 (`m1_runner.deployed_scat32`) via
  `oed_design_v4.info_matrix_full` + `subspace_from_fixedstar`; palette =
  `oed_design_v4.palette4(2000, fast=True)`; fixed-dose rows =
  `oed_design_v5.fixed_dose_scat32(32)`.
- **Trajectory capture.** Every `guard_eval` call during the descending scan
  `m = 972 … 0` is recorded by wrapping `oed_design_v5.guard_eval` (its only
  caller is `mixture_search`); `α = m/972`. Wall time 754 s (< 15 min budget).
- **Result (frozen in the cache).** `mixture_status = MIXTURE_FAIL:effD`,
  `PATH_FEASIBLE_ALPHA = None`, verdict `NO_FEASIBLE_POINT_ON_PATH`. Dose
  deviation is 2–11× over the 0.05 band for every `α > 0`; the spectral floor
  `μ_min` drops below `0.5` by `α ≈ 0.06`; the A-risk ratio never exceeds its
  `1.05×` cap. The only dose-feasible point (`α = 0`, balanced) fails the
  separate efficiency floor `effD ≥ 0.95`. This is consistent with (and
  stronger than) the R17 consultation evidence
  (`results/round63_m1/R17_CONSULTATION_EVIDENCE.md`).
- **No selection**: single predeclared DEV scene / anchor; whole trajectory
  plotted.

### (b) `actiii_b.pdf` — constructive dose-only primal lower bounds
- **Source.** `results/round63_m1/R18_GAP_PROBE_REPLICATION.md`, the frozen
  12-cell replication table (six DEV families × two anchors), read verbatim
  (`DOSE_ONLY_PRIMAL_GAP_PER_R`). Right axis = D-efficiency lower bound
  `exp(gap)`.
- **No selection**: all six families and both anchors shown. Development-only
  lower bound; the construction is support-preserving (existence, not the
  dose-only optimum) per R18 §1.2.

### (c) `actiii_c.pdf` — descriptive full-stack three-status distribution
- **Source.** `results/round63_m1/shards/M1_CERT_*.csv` (480 rows). Parsed
  with the overflow-merge reader (`_read_cert_csv`, mirroring
  `code/round63/m1_score.py`) because the `solver_status_primary` /
  `solver_status_retry` fields embed commas.
- **Left panel.** Terminal status counts by the four `(ν, b)` anchors
  (120 cells each): 299 COUNTEREXAMPLE, 181 NUMERICAL_UNRESOLVED, 0 CERTIFIED.
- **Right panel.** ECDF of the finite `primal_gap_lower_per_r` values for the
  299 COUNTEREXAMPLE cells (range 1.19–1.87, median 1.43); the
  `NUMERICAL_UNRESOLVED` cells carry `-inf` and are excluded from the ECDF.
  Threshold line at `G_stack/r ≤ 10^{-2}`.
- **No selection**: descriptive branch, every cell shown.

### (d) `actiii_d.pdf` — ridge-vs-0.60 fixed-dwell result (CENTREPIECE)
- **Dot strip.** Per-image `dQ = PSNR_rad(RIDGE-SCAT32) − PSNR_rad(SCAT32-060)`
  at `ν = 2000`, **seed-mean over the five frozen seeds**, from the imaging
  shards. Arm identity = shard-id prefix (`M1_<ARM>_<idx>`), **not** the CSV
  `arm` column (which holds the estimator name `RQL`). Campaign median
  `1.867 dB`; 19/24 scenes `> 0`; contour negatives shown (nothing hidden).
- **Thumbnails.** GT | SCAT32-060 | RIDGE-SCAT32 reconstructions for three
  disclosed cells at **seed 0, ν = 2000**, each annotated with its `PSNR_rad`
  (seed-0 realization) and a per-row `dQ` badge. Generated by the frozen
  campaign cell machinery (`campaign.run_cell` via `m1_runner.m1_cell`); the
  reconstruction is the `select_eta.select_eta_and_fit` output captured
  in-path.
- **Selection rule (disclosed cells).**
  - median-closest → **`m1_chirp_3`**. The campaign median 1.867 dB is the
    midpoint of the 12th (`m1_chirp_3`, +1.539) and 13th (`m1_spokes_3`,
    +2.195) order statistics, so both are exactly equidistant (0.328 dB) from
    the median; the tie is broken toward the lower order statistic
    (`m1_chirp_3`).
  - best/max → **`m1_maze_1`** (max dQ).
  - worst/min → **`m1_contour_1`** (min dQ — a contour, as expected).
- **Confirmatory-lock note.** The `imageset="m1"` confirmatory guard requires
  `M1_FREEZE_LAUNCHED`; the recon precompute set `M1_FREEZE_LAUNCHED=1`. The
  campaign is already unblinded (`M1_VERDICTS.json`, tag `m1-freeze @ 6f00932`,
  HEAD `3ea6306` = "M1 UNBLINDED"); this is **post-unblinding display
  generation only** — no confirmatory scene, count, or endpoint is recomputed
  or reopened. RIDGE-SCAT32 confirmatory ridge calibrations were computed and
  cached on first call under `results/round63_m1/designs/ridge_scat32_m1_*.npz`.

### `fig_mechanism.pdf` — hero mechanism figure
- **Drawing** for the pipeline and the dead-time timeline (illustrative
  SCAT32 pattern, fixed RNG seed), sharing the paper-1 mechanism figure's
  visual language (`paper/figs/fig_mechanism_p1.*`): orange beam arrows, blue
  bucket-counts path, insets inside the boxes, labels below, green RQL box.
  The object / reconstruction insets are real campaign data: the confirmatory
  scene `m1_maze_1` and its RIDGE-SCAT32 seed-0 reconstruction from
  `results/round63_m1/ACTIII_D_RECON.npz` (deterministic synthetic fallback
  if the cache is absent). The global power knob is drawn as a dial (halo +
  ticks + needle) attached to the source — the paper's single control.
- **Ridge curve (data).** `J(ρ̄; ν)` is evaluated **densely (130 points,
  0.3 ≤ ρ̄ ≤ 30)** from the frozen exact count-law machinery
  (`code/round63/physics.fisher_exact`, information about log-λ per unit
  time — the same definition as `fisher_ridge.py`), instead of the coarse
  49-point `fisher_ridge.csv` cache, whose ν=200 tail (ρ̄ ≥ 44) carries a
  visible finite-difference artifact near the count ceiling; the evaluation
  range ends at ρ̄ = 30, below the artifact onset (ρ̄ ≈ 31 at ν = 200), so
  both curves are smooth end to end. The ρ̄* markers (22.2 @ ν=2000, 10.0 @
  ν=200) are the frozen values from `fisher_ridge.json`. The displayed law
  `ρ̄*(ν) = (6ν)^{1/3} − 2/3` is the **zero-jitter limit (c = 0)** — annotated
  in-panel so it cannot be misread as the general jitter-capped law.

## Ready-to-paste Chen-style captions

Noun-phrase label; semicolon symbol glossary; exhaustive sub-panel
enumeration; per-panel metrics bound with "respectively"; housekeeping lines
where relevant. Metric values are those frozen in the figures.

**Fig. M (hero).** Schematic of the dead-time-aware single-pixel imaging
pipeline and the count-information ridge law. (a) Imaging pipeline: the global
power knob Φ (dial) is the single scalar source multiplier setting the
detector load ρ̄ = τ E[λ]; the beam (orange arrows) passes from the laser
source through the DMD, which displays the SCAT32 scattered-sparse
illumination patterns a_i (inset), to the object x (inset: the confirmatory
maze scene), and the SPD photon-counting detector feeds the bucket counts N_i
(blue arrow) to the RQL reconstruction x̂ (inset: the RIDGE-SCAT32
reconstruction of the same scene, seed 0, ν = 2000). DMD: digital micromirror
device; SPD: single-pixel (bucket) detector; RQL: renewal quasi-likelihood
reconstruction. In the photon-arrival timeline (lower inset), recorded counts
(green markers) each open a non-paralyzable dead-time hold of length τ
(shaded; one hold is jitter-broadened, darker), and arrivals inside a hold are
lost (×); the exposure is T = ντ, so ν = T/τ is the dwell budget.
(b) Count-information efficiency J(ρ̄; ν) versus detector load, evaluated
from the exact finite-window count law for ν = 2000 and ν = 200, whose
optimal-load ridges are ρ̄* = 22.2 and ρ̄* = 10.0, respectively; the ridge
follows the finite-window law ρ̄*(ν) = (6ν)^{1/3} − 2/3 (zero-jitter limit,
c = 0), and the global power knob (arrow) moves the operating point onto the
ridge. The color is used only for display purposes.

**Fig. A (a).** Guard collapse of the adaptive OED-DT mixture along the
descending exact-rounding path for the development glyph scene at the fast
anchor (ν = 2000, b = 0.60); development-only evidence, not a confirmatory
endpoint. The dose deviation max|d/d̄ − 1| (orange), the spectral floor μ_min
(green), and the A-risk ratio tr(V^{-1})/tr(V_fix^{-1}) (blue) are shown versus
the path-specific OED fraction α = m/972, where m is the number of retained
scene-adapted rows, V is the task-subspace information matrix, and V_fix is the
load-matched SCAT32 reference; the dashed guard bars are the ±5% dose band
(0.05), the spectral floor (0.5×), and the A-risk cap (1.05×), respectively.
Every adaptive fraction α > 0 exceeds the dose band by 2–11× and, by α ≈ 0.06,
breaches the spectral floor, while the A-risk cap never binds; only the
balanced endpoint α = 0 is dose-feasible and it fails the separate efficiency
floor effD ≥ 0.95, so PATH_FEASIBLE_ALPHA = None (NO_FEASIBLE_POINT_ON_PATH).

**Fig. B (b).** Constructive dose-only geometry headroom over the six
development families; development-only lower bound, not a confirmatory endpoint
or population estimate. For each family the dose-only primal gap per task
dimension, [log det V(ξ) − log det V_SCAT32]/r within the ±5% per-pixel dose
band, is shown at anchor A (ν = 2000, b = 0.60; circles) and anchor B
(ν = 200, b = 0.05; diamonds), and the right axis reports the D-efficiency
lower bound exp(gap). Across glyph, chirp, maze, spokes, contour, and
microtexture the gaps are 0.78, 0.86, 0.59, 0.90, 0.90, and 0.92 at anchor A
and 0.90, 0.50, 0.51, 0.79, 0.59, and 0.72 at anchor B, respectively
(D-efficiency lower bounds 1.6–2.5×). The construction is support-preserving
and establishes the existence of headroom, not the dose-only optimum.

**Fig. C (c).** Descriptive full-stack three-status distribution under the
fallback-descriptive branch (0/480 CERTIFIED). (a) Terminal status of the 480
certificate cells split by the four (ν, b) anchors (120 cells each): at
(ν200, b0.05), (ν200, b0.60), (ν2000, b0.05), and (ν2000, b0.60) the cells are
77, 67, 84, and 71 COUNTEREXAMPLE and 43, 53, 36, and 49 NUMERICAL_UNRESOLVED,
respectively, with no CERTIFIED cell. (b) Empirical cumulative distribution of
the finite primal gap per dimension for the 299 COUNTEREXAMPLE cells (median
1.43); the dashed line is the near-optimality bar G_stack/r ≤ 10^{-2}, which
every counterexample exceeds by about two orders of magnitude. The certificate
is reported descriptively and no categorical near-optimality claim is made.

**Fig. D (d).** Fixed-dwell reconstruction quality of the global power knob:
RIDGE-SCAT32 against balanced SCAT32-060 at ν = 2000, seed 0. (a)–(c) the
median-dQ scene (m1_chirp_3): ground truth, SCAT32-060 (ρ̄ = 0.60), and
RIDGE-SCAT32 (ρ̄* ≈ 22), with radiometric PSNR_rad of 9.4 dB and 10.9 dB for
the two reconstructions, respectively (dQ = +1.5 dB); (d)–(f) the best-dQ scene
(m1_maze_1), PSNR_rad of 28.7 dB and 35.4 dB, respectively (dQ = +6.7 dB);
(g)–(i) the worst-dQ scene (m1_contour_1, a contour), PSNR_rad of 13.6 dB and
7.7 dB, respectively (dQ = −5.8 dB). (j) Per-image dQ (seed-mean over five
seeds) for all 24 confirmatory scenes grouped by family; the solid line is the
campaign median 1.87 dB, the dashed line the 1.0 dB operating bar, and 19/24
scenes exceed zero. PSNR_rad denotes the radiometric peak signal-to-noise
ratio and dQ = PSNR_rad(RIDGE-SCAT32) − PSNR_rad(SCAT32-060); the thumbnail
values are the seed-0 realization. The color is used only for display purposes,
and each row shares a common grayscale scale.

**Fig. E (e).** Conjugate-corners operating doctrine for the global power knob.
The plane is spanned by the optical-power/flux budget (horizontal) and the
acquisition-time budget (vertical). In the photon-budgeted corner (low power,
ample time; Q90 elapsed-time secondary) the SCAT32-SAFE comparator is used and
the corrected preregistered elapsed-$T_{\rm opt}$ speed verdict passes, whereas in
the time-limited, power-available corner (ample power, little time; fixed-dwell
quality primary) the ridge operating point ρ̄* delivers a
+1.87 dB median gain, and RIDGE_OPERATING_PASS = TRUE; the curved arrow marks
the single global power knob that moves the system between the two conjugate
corners. The color is used only for display purposes.
